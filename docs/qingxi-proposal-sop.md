# 青曦標準提案 SOP

> 內部私有資產。此文件用於面談、提案、報價說明與簽約前後的節奏控制，不對外公開全文。

## 目標

- 不跟業主只比總價，而是先建立「流程理解 + 隱蔽工程理解 + 管理價值」。
- 把公開版官網內容與內部提案內容分層，避免高價值資產過早外流。
- 讓 `quote-studio`、`contract-studio`、`standard-sop`、`renovation-process` 四者在敘事邏輯上保持一致。

## 使用時機

### A. 初次接洽前

- 引導對方先看公開版頁面：`/standard-sop`
- 引導先填條件：`/tools`、`/requirement-form`
- 目標不是直接成交，而是先篩出高意圖與低意圖

### B. 第一次諮詢 / 線上會議

- 用「合作流程圖」先講整體節奏
- 再用「公版老屋翻新甘特圖」講排程與時程變數
- 只講到公開層級，不進入完整樹狀解剖細節

### C. 現場丈量後 / 正式提案前

- 帶入個案風險盤點
- 開始使用「樹狀工程解剖圖」說明報價為什麼不是只看表面材料
- 若對方只想比價，優先強調隱蔽工程、工序管理與返工成本

### D. 簽約前

- 用 `quote-studio` 的報價分類對齊樹狀工程解剖圖
- 用 `contract-studio` 的付款節點對齊甘特圖與查驗里程碑
- 確認業主已理解：付款節點不是任意拆分，而是跟工程節點綁定

| 層級 | 用途 | 可否公開 | 載體 |

## 面談標準節奏

### 第 1 段：先講流程，不先講價格

- 先用 5 步驟合作流程圖建立節奏感
- 目標：讓對方知道青曦不是隨口估價，而是有完整前置判斷與排程邏輯

### 第 2 段：再講時程，不先講材料

- 用甘特圖說明：
  - 為什麼老屋翻新不能只問幾天做完
  - 為什麼隱蔽工程與設備排產會影響時程
  - 為什麼完工尾款要放在點交之後

### 第 3 段：最後才進入樹狀工程解剖圖

- 用來處理以下情境：
  - 對方覺得報價比工班高
  - 對方只比較表面材價格
  - 對方不理解管理費、監工費、工序整合的價值

## 樹狀工程解剖圖

```text
裝修工程
├─ 前置保護與拆除
│  ├─ 公共區保護
│  ├─ 室內保護
│  ├─ 拆除工程
│  └─ 清運與素地整理
├─ 管線與設備預留
│  ├─ 強電與弱電
│  ├─ 給排水
│  ├─ 空調與設備預埋
│  └─ 智能家居預留
├─ 泥作與防水基礎
│  ├─ 打底與修補
│  ├─ 濕區防水
│  └─ 貼材前基準面校正
├─ 木作與櫃體整合
│  ├─ 天花與封板
│  ├─ 系統櫃量測與下單
│  └─ 油漆與表面收邊
└─ 設備與完工交付
   ├─ 衛浴與家電安裝
   ├─ 玻璃 / 五金 / 門窗
   ├─ 功能測試
   └─ 驗收與點交
```

## 樹狀工程解剖圖的講法

### 一句話版本

- 「你看到的是表面完成，我們管理的是整個底層工序與風險。」

### 進階版本

- 「很多報價單只列得出最後看得到的材料，但真正決定是否耐住、是否返工、是否追加的，常常是拆除、水電、防水、泥作、設備預留與工序銜接。」

### 收斂版本

- 「如果別人的報價更低，先不要只比總價。先比他有沒有把這些隱蔽工程節點列清楚，因為很多便宜其實只是把未來風險先藏起來。」

## 報價與付款節點的標準說法

### 報價說法

- `quote-studio` 的分類就是提案解剖圖的對照骨架
- 報價說明順序固定為：
  1. 前置保護與拆除
  2. 管線與設備預留
  3. 泥作與防水
  4. 木作、櫃體與表面整合
  5. 設備安裝、測試與交付

### 付款節點說法

- 簽約訂金 `30%`：檔期保留、前置備料與施工排程啟動
- 工程開工 / 中期款 `30%`：水電管線配置完成，並完成主要隱蔽工程查驗
- 設備 / 櫃體進場款 `30%`：現場尺寸穩定，系統櫃、設備或主要面材排產 / 進場前
- 完工尾款 `10%`：完工通知點交後支付，若有小瑕疵則依保留款機制處理

## 陽宅風水評估 / 化解方案的使用原則

- 不作為對外首頁主賣點
- 不在初次接洽時主動完整攤開
- 視案況作為加值型提案亮點
- 用法重點是「補強空間判斷與生活舒適度邏輯」，不是走玄學話術

## 禁止事項

- 不要一開始就把所有私有細節講光
- 不要跳過流程圖，直接用複雜圖表壓對方
- 不要用抽象形容詞取代具體風險說明
- 不要把付款節點講成單純收款規則，必須綁查驗里程碑
- 不要讓樹狀工程解剖圖與 `quote-studio` 分類脫鉤
